What is AAA Testing?

Basically, it recommends separating your test method into three sections: arrange, act and assert. Each one of these are only accountable for the portion they are named after.

In the arrange section, included is the code needed to setup that test. Objects are be created, if using mocks then they are setup and possibly expectations are set. After this come the Act, which is the call of the method being tested.

Then comes Assert, where this includes checking whether the expectations were encountered. Using this process makes the code well structured and simple to understand.
