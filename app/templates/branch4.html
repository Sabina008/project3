What is SOLID?

SOLID is an acronym which stands for 5 design principles in Object Oriented Programming:

S - Single-Responsibility Principle

O - Open-Closed Principle

L - Liskov Substitution Principle

I - Interface Segregation Principle

D - Dependency Inversion Principle

Single-Responsibility Principle

Each module should have accountability over a single part of the functionality provided by the software.

We can see an example of this in the screenshot referenced from my calculator project.

The screenshot is from my calculator project, the code reflects single responsibility of a class method and a static method.

Open-Closed Principle

Objects or entities should be open for extension but closed for modification.

Software entities (classes, modules, functions, etc.) should be open for extensions, but closed for modification.

Liskov Substitution Principle

Liskov Substitution Principle means each subclass or derived class should be substitutable for their base or parent class.

We can see a perfect example referenced from my calculator project. Each derived class (Child Class) is substitutable for the base.

Interface Segregation Principle

This principal state that no client should be forced to depend on methods it does not use.

In other words, you should not add further functionality to an existing interface by adding new methods.

An example is included in the screenshot which shows no additional functionality was included to an existing interface.

Dependency Inversion Principle

This principal is a way to decouple software modules.

This principle states that:

- High-level modules should not depend on low-level modules. Both should depend on abstractions.

- Abstractions should not depend on details. Details should depend on abstractions.
