What are the principles of Object-Oriented Programming: Encapsulation, Inheritance, Polymorphism, and Abstraction?
Encapsulation

This is one of the essential concepts in object-oriented programming (OOP). It defines the idea of grouping data and methods that work on that data within a single unit. This idea is frequently used to hide the internal state, of an object from the outside.

Inheritance

The main benefit of inheritance is it offers code re-usability. Instead of writing identical code, you can simply get the properties of one class into the other which in return will reduce time and money.

We can see an example of this in the screenshots included from my calculator project. You can see in line 1 in each python file they are all similar, the main difference being the Parent and Child Class.

Polymorphism

This is one of the core concepts of object-oriented programming languages that permits a precise routine to use variables of different types at different times. It describes the concept that objects of different types can be retrieved through the equivalent interface.

Abstraction

This is the notion of object-oriented programming that displays only vital qualities and hides pointless data. The key purpose is hiding needless data from users. Abstraction is picking data from a bigger pool to show only applicable information of the object to the user. This assists in decreasing programming complication and efforts.
